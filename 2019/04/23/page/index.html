元星科技二面-严强福-产品经理

Apr 23, 2019

一、你最喜欢的互联网产品是什么

答：我最喜欢的互联网产品是微信读书，我将从微信读书的定位、用户场景以及核心功能进行分析

1.定位：社交+阅读

微信读书是依托于微信产生的产品，所以有天然的社交链，而正是因为这个，我们可以看到好友正在看的书，以及好友看完书写的想法，可以进行思想上的碰撞，并且可以看到好友的阅读时长，激励我去进行阅读

2.用户场景

分为三个场景：看书前、看书时、看书后

2.1 看书前

2.1.1 通过分析我以往的阅读记录推送我喜爱的书籍
2.1.2 读书日话题书籍推荐
2.1.3 好友正在阅读书籍
2.1.4 刚加入微信阅读时赠送一本书
2.1.5 书籍购买较为便宜，以及有月卡、季卡的推出 ，并且通过分享给好友获得

2.2 看书时

2.2.1 良好的阅读体验，支持EPUB和TXT格式，排版良好
2.2.2 可以看到他人对于某个段落的想法
2.2.3 可以选择听书
2.2.4 自己可以做好笔记，写好想法

2.3 看书后

2.3.1 根据对应时长获得书币，用于购买图书
2.3.2 在想法社区中阅读自己的想法以及他人的想法以便对书籍复习
2.3.3 与好友的阅读时长进行对比
2.3.4 好友点赞和评论实现交流以及自我实现感

3.功能分析

3.1核心功能优缺点分析

微信读书的核心功能是发现添加书籍（根据各类分类选择书籍），阅读，发表想法，听书，分享书籍，赠送书籍，兑换书币。主要会将发现添加书籍，阅读书籍，发表想法，三个流程图展示在下面。

3.1.1 发现添加书籍

优点：用户加入书架的入口众多，并且当用户免费分享和增一得一成功后微信读书会自动将书籍加入用户的书架。
缺点：加入书架之前应当有个是否将这本书加入书架的提醒或者是将这本书加入哪个分组，因为从这个流程来看添加至书架和书单没有任何联系，所以当用户创建好书单后，添加书籍却不能直接添加至自己的书单，而是要在书架里移来移去，那么书单功能显的很鸡肋。

3.1.2 阅读书籍

优点：阅读书籍很简单，赠一得一和免费分享等功能即使不分享，也可以阅读，虽然只是一部分，但是很好的满足了用户先去阅读，随后引诱用户购买。
缺点：在老版本这样的阅读记录不明显，不利于用户查找，但最新版本在这方面进行了更新，用户可以查看自己的阅读记录

3.1.3发布想法

优点：简单高效，用户可以直接在想法里发布想法，不用看书不用选择什么，直接就像朋友圈一样发布就行，也可以选择一本书或者在阅读的时候选择一段文字发布自己的想法，并且可以取决私密还是公开
缺点：为书籍打分不明显，在最上方，以及在公开和私密也不明显

3.2 主要功能分析

功能分析主要通过上面所提到的发现添加书籍（根据各类分类选择书籍），阅读，发表想法，听书，分享书籍，赠送书籍，兑换书币进行分析
3.2.1发现
“发现”功能放在一级入口，是微信读书通过好友阅读来引导好友打造一个阅读朋友圈，而关于发现书籍功能，前面的流程图也提到了优缺点，如果单独发现板块来讲，在交互设计方面使用的左右滑动的卡片形式，界面简洁，除此之外，发现板块还会有用户每周的阅读报告，时长多少，排名多少，给我一个总结，并且附上微信昵称签名，签名这个细小的设计让我很感动。

3.2.2阅读
阅读界面的选项有：目录、书签、划线、进度调节、亮度调节、背景更换、字体以及大小选择、书写书评、查看书评、今天一起阅读本书的人、设置为私密阅读、充值购买以及右上角的菜单按钮，里面包括：赠送书籍、点评书籍、分享、添加书签、语音朗读、纠错以及书籍详情等。
如果在阅读过程中长按一段文字则会复制划线，写想法，查询，分享，纠错。
总的来说基础功能微信读书都具备，并且各大app也相差不远，但是在界面的设计上微信读书更加简洁舒服。

3.2.3发表想法
很明显的可以感觉到微信读书在很用力的引导我写书评，不管有没有阅读都可以写，并且添加想法的按钮也非常显眼，所以对于微信读书来说，一直都是在打造读书朋友圈。
而这也是对于好友阅读书籍的一种帮助，相对于哪些高高在上以及一些网站的评分而言，自己相同爱好的人的评分和想法显得更加可以相信，所以我可以更加放心的阅读书籍，并且发表想法跟朋友交流。

3.2.4兑换书币
这个功能是我认为比想法还要更加亮点的功能，让我读书后使用自己的读书时长换取书币然后再利用书币去买书，给用户一个双重的荣誉感，并且在读书时长上人总是有一种攀比心理，在这种心理的驱使下，我会使用更多时长在微信读书上。

4.分析微信读书的吸引我的的地方

4.1分享：分享和奖励是分不开的，用户在赠一得一和免费领取书籍上都是通过分享，用户只要分享成功，就可以免费获得一本书，而在赠一得一的设计上，我分享给好友后，好友领取后，用户也可以获得一本书籍，这样就可以吸引一批用户前来使用。
4.2激励：前面提到的时长问题，用户总是会互相攀比的，当看到好友的时长比我长的时候总会忍不住去攀比。
4.3奖励：时长的设计，阅读的越多，那么就可以兑换更多的书币，就可以买更多的书籍，并且对于用户来说，并且除去这个奖励机制以外，用户发表想法后，得到的赞和评论都是一种奖励。

5．产品总结及建议

微信读书已经养成了我一个读书习惯，并且其中的奖励机制是我特别喜欢，但是微信读书依旧有许多值得改进的地方。下面通过SWOT行分析
S：前面提到的背靠微信大山，自带的流量，自带的好友体系，自带的极大用户群，对于一个想做读书朋友圈的app来说占据了天时，并且微信读书的开发人员善于听取用户的意见。

W：网络文学较少，并且市面上已经有了许多其他的阅读类app，以及在部分功能上微信读书显得鸡肋，书单功能便是如此；商业化方面也较为单一

O：我认为微信读书完全可以跟微信一样，打造一个老年人使用的读书软件，在上面的数据中显示60%是年轻人，而老年人占得用户量极少，所以我认为微信读书可以有一波人口红利，并且已经有了听书作为背景完全可以做

T：依旧还是市面上其他app的竞争，首先qq阅读和微信阅读一样是拥有微信登陆的，并且qq阅读拥有的书籍更多。

二、尽可能细节化告诉我当点击一个app到出现这个app正式页面的过程，这个app会发生什么

答：

1.App页面过程

1.1点击手机app图标，页面展示app的启动页面，有些是广告，有些是app的slogan，有些是图片。
1.2 展示app的主页面，但是当网络不好时，有些app会展示骨架加载图（比如知乎 简书等app）

###2.App原理过程（这方面是当年学代码时看过他人写过的答案）
2.1.点击手机app图标，Launcher进程采用Binder IPC向system_server进程发起startActivity请求；
2.2.system_server进程接收到请求后，向zygote进程发送创建进程的请求；
2.3.Zygote进程fork出新的子进程，即App进程；
2.4.App进程，通过Binder IPC向sytem_server进程发起attachApplication请求；
2.5.system_server进程在收到请求后，进行一系列准备工作后，再通过binder IPC向App进程发送scheduleLaunchActivity请求；
2.6.App进程的binder线程（ApplicationThread）在收到请求后，通过handler向主线程发送LAUNCH_ACTIVITY消息；
2.7.主线程在收到Message后，通过发射机制创建目标Activity，并回调Activity.onCreate()等方法。
2.8.到此，App便正式启动，开始进入Activity生命周期，执行完onCreate/onStart/onResume方法，UI渲染结束后便可以看到App的主界面。

三、如何解决技术开发人员和UI设计人员意见不一致的情况？

答：首先产品比技术开发会更懂UI和交互，比UI设计人员更懂技术研发，这是产品的相对优势。

  1. 寻求导致意见不一致的本质原因，与两者进行单独沟通；是因为技术没有按照UI所做的切图标注进行开发，还是UI设计人员所提的要求没有必要实现，并且记录下来
  2. 将两者的问题与产品战略放到一起，分清楚事情优先级，
  3. 弄清楚研发和设计手头上的工作，并且进行衡量下两者的工作量
  4. 做好这方面的工作后思考如何让两者达到一个平衡点并进行说服，并且让两者明白哪个更为重要，是对产品体验更加好的方案
  5. 在工作上强调团队关系，打破工种关系
